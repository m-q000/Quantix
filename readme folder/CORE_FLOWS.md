# Core Flows

## 1. Stall Registration Flow
1. Vendor registers
2. Submits application
3. Admin reviews
4. Assigns location + rules
5. Approves
6. QR generated
7. Stall becomes active

## 2. Inspection Flow
1. Officer scans QR
2. System retrieves stall
3. System checks:
   - Location match
   - Time validity
   - Activity match
   - Subscription status
4. Officer records:
   - OK / Warning / Violation

## 3. Citizen Browsing Flow
1. Open map
2. Filter stalls
3. View details
4. Check if open

## 4. Expiry Flow
1. System detects expiry
2. Vendor notified
3. If not renewed:
   - Status → expired
   - Inspection shows violation
